# Архитектура персонального ассистента

## Обзор
Персональный офисный ассистент разработан для поддержки члена совета директоров в выполнении управленческих задач. Архитектура построена на принципах Personal AI Infrastructure (PAI) с акцентом на сохранение контекста, обучение на взаимодействиях и предоставление релевантной поддержки в реальном времени.

## Контекст
Ассистент работает в локальном окружении пользователя и имеет доступ к:
- Файлам пользователя в директории `USER/` (миссия, цели, проекты и т.д.)
- Системным файлам в директории `SYSTEM/` (навыки, конфигурация, документация)
- Внешним инструментам через навыки (по запросу и с разрешения пользователя)

## Ключевые компоненты

### 1. TELOS (Глубокое понимание целей)
Расположен в `USER/` и содержит:
- MISSION.md - жизненная и профессиональная миссия
- GOALS.md - текущие краткосрочные и долгосрочные цели
- PROJECTS.md - активные и архивные проекты
- BELIEFS.md - ключевые убеждения и принципы
- MODELS.md - ментальные модели и подходы к решению проблем
- STRATEGIES.md - стратегии достижения целей
- NARRATIVES.md - личные истории и идентичность
- LEARNED.md - извлеченные уроки и инсайты
- CHALLENGES.md - текущие сложности и препятствия
- IDEAS.md - будущие идеи и возможности

### 2. Система навыков
Расположена в `SYSTEM/skills/` и содержит модульные capabilities для выполнения specific задач, таких как:
- Подготовка материалов к совещаниям
- Анализ финансовых и стратегических документов
- Планирование и организация работы
- Подготовка презентаций и докладов
- Транскрибация и summarization встреч
- Исследование и сбор информации

### 3. Система памяти
Отслеживает все взаимодействия для непрерывного обучения и улучшения ответов.

### 4. Хуковaya система
Реагирует на события lifecycle (старт сессии, завершение задачи и т.д.) для автоматической загрузки контекста, уведомлений и т.д.

### 5. Конфигурация
Хранится в `SYSTEM/config/` и включает настройки персонализации (имя ассистента, предпочтения и т.д.).

## Потоки данных
1. Пользователь делает запрос через интерфейс OpenCode
2. Ассистент загружает релевантный контекст из `USER/` (TELOS, недавние взаимодействия)
3. Ассистент определяет, какие навыки из `SYSTEM/skills/` применить
4. Ассистент выполняет задачу, используя выбранные навыки и контекст
5. Результат возвращается пользователю
6. Взаимодействие сохраняется в системе памяти для будущего обучения
7. При необходимости обновляются файлы в `USER/` (например, LEARNED.md или PROJECTS.md)

## Технологии и зависимости
- Базируется на OpenCode/Claude Code engine
- Использует Bash и TypeScript для навыков и хуков
- Хранит данные в обычных текстовых файлах (Markdown, JSON)
- Не требует внешних сервисов для базовой работы (можно расширить)

## Нефункциональные требования и ограничения
- Конфиденциальность: все данные хранятся локально
- Контроль пользователя: пользователь решает, какие данные делиться и какие инструменты использовать
- Прозрачность: пользователь всегда может увидеть, какие файлы используются и изменяются
- Переносимость: вся информация о пользователе хранится в `USER/` и легко резервируется/перемещается
- Безопасность: чувствительные данные (API keys и т.д.) хранятся в `.env` и исключены из git

## Roadmap

2. Разработка базовых навыков для офисных задач
3. Интеграция с внешними сервисами (по запросу)
4. Улучшение системы памяти и персонализации
5. Добавление голосового интерфейса и уведомлений
